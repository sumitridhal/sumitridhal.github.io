These notes pick apart motion from [Object & Archive](https://objectandarchive.com/) as **five standalone recipes**—each maps to one live preview below (Part 3 mirrors the **`/sections`** scroll stack in code). Together they cover **CSS `@keyframes`**, **GSAP timelines**, **ScrollTrigger** with **Lenis** on the document scroller, and **nested Locomotive Scroll** for embed-local smoothing plus **CSS variables**, plus a **layout-only** hover pattern with **no image tweening**. This article is complete as-is; there is no “Part 6” appendix.

## How to read a recipe

| Label       | Meaning                                                                 |
| ----------- | ----------------------------------------------------------------------- |
| **Target**  | The node or CSS property that actually changes.                        |
| **Driver**  | What kicks the change: load, click, scroll, hover/focus, etc.          |
| **Runtime** | What interpolates: CSS animation, GSAP, native scroll, or DOM updates. |

## Part 1 — Hero curtain and split reveal

**Target:** A full-bleed **curtain** over a split hero; **driver** is first paint (then optional replay); **runtime** is **CSS** only. `WritingObjectArchiveHeroRevealPreview` toggles classes (`is-lifting` → `is-open`) so the curtain runs `writing-oa-hero-curtain-up` once; reduced motion snaps the curtain off and disables replay.

Curtain rules and keyframes live under `.writing-oa-hero*` in `_writing.scss`:

```scss
.writing-page__body .writing-oa-hero__curtain.is-lifting {
  animation: writing-oa-hero-curtain-up 1.28s cubic-bezier(0.33, 0, 0.2, 1) forwards;
}

.writing-page__body .writing-oa-hero__curtain.is-open {
  transform: translate3d(0, -100%, 0);
}

@keyframes writing-oa-hero-curtain-up {
  from {
    transform: translate3d(0, 0, 0);
  }
  to {
    transform: translate3d(0, -100%, 0);
  }
}
```

## Part 2 — Left column: stacked plates and scale ladder

**Target:** Sixteen centered `<img>` plates, **uniform `scale` only**. **Driver:** **Next →** / **← Back** stepping `advanceCount`. **Runtime:** one **GSAP `timeline`** per step; every layer tweens in **parallel** at label time `0` (`power2.inOut`, duration `0` when reduced motion).

Core pattern in `WritingObjectArchiveLeftPanelPreview`:

```ts
const tl = gsap.timeline({
  defaults: { duration: dur, ease: EASE },
  onComplete: () => {
    if (alive) setIsAnimating(false)
  },
})

imgs.forEach((img, j) => {
  tl.to(img, { scale: targetScale(j, advanceCount, n) }, 0)
})
```

`targetScale` maps each index to **passed** (full-bleed), **focal** (mid), or **ahead** (hidden) scale constants.

## Part 3 — Composite: viewport Lenis + scrubbed section motion (`/sections`)

**Target:** Several layers pick up **`transform`** from **`gsap.fromTo`** with **`scrub: true`**: hero **`#section-1`** (**`yPercent` `0 → 20`**), obsessions wrapper **`.obsessions__scroll-shift`** (**`y` `120 → 0`**), and full-bleed panel **`<img>`** backgrounds in sections **3, 5, and 7** (**`translate(0%, 25%)`** via **`yPercent: 25`**). **Driver:** vertical scroll (smooth Lenis path or native when reduced). **Runtime:** **Lenis** stepped from **GSAP’s ticker**, **`ScrollTrigger.scrollerProxy(document.documentElement)`** so scrubbed progress matches what Lenis shows, wrapped in **`gsap.context`** for clean teardown.

Live stack: **[`/sections`](/sections)** — [`SectionsPage.tsx`](src/pages/SectionsPage.tsx) + [`useSectionsScrollExperience.ts`](src/hooks/useSectionsScrollExperience.ts).

### Lenis and ScrollTrigger on the document scroller

[`LenisProvider`](src/providers/LenisProvider.tsx) builds Lenis with **`autoRaf: false`**; [`lenisService`](src/services/lenisService.ts) calls **`lenis.raf`** from **`gsap.ticker`**. [`useLenisScrollTrigger`](src/hooks/useLenisScrollTrigger.ts) keeps **`ScrollTrigger`** aligned with smooth scroll: on each Lenis **`scroll`** event it calls **`ScrollTrigger.update()`**, and a **`document.documentElement`** proxy maps programmatic **`scrollTop`** reads/writes to **`lenis.scroll`** / **`lenis.scrollTo`**:

```ts
// useLenisScrollTrigger.ts (trimmed)
lenis.on('scroll', () => ScrollTrigger.update())

ScrollTrigger.scrollerProxy(document.documentElement, {
  scrollTop(value) {
    if (typeof value === 'number') {
      lenis.scrollTo(value, { immediate: true })
    }
    return lenis.scroll
  },
  getBoundingClientRect() {
    return {
      top: 0,
      left: 0,
      width: window.innerWidth,
      height: window.innerHeight,
    }
  },
  pinType: 'fixed',
})
```

### Hero parallax: section 1 driven by section 2

The hero scrubs **`yPercent`** from **`0`** to **`20`** (positive = downward in GSAP) while **`#section-2`** moves from entering the viewport until **its top reaches the viewport top**—motion **stops** once that range completes (matching the hook comment on **`useSectionsScrollExperience`**).

```ts
// useSectionsScrollExperience.ts (trimmed)
gsap.fromTo(
  section1,
  { x: 0, yPercent: 0 },
  {
    x: 0,
    yPercent: 20,
    ease: 'none',
    scrollTrigger: {
      trigger: section2,
      start: 'top bottom',
      end: 'top top',
      scrub: true,
    },
  },
)
```

### Obsessions wrapper lift

**`.obsessions__scroll-shift`** wraps the obsessions column and scrubs from **`translate(0, 120px)`** toward **`0`** while section 2 is approaching the top edge:

```ts
gsap.fromTo(
  obsShift,
  { x: 0, y: 120 },
  {
    x: 0,
    y: 0,
    ease: 'none',
    scrollTrigger: {
      trigger: section2,
      start: 'top bottom',
      end: 'top 30%',
      scrub: true,
    },
  },
)
```

### Panel background `<img>` parallax (sections 3, 5, 7)

Panels use an absolutely positioned cover **`<img>`** under scrims/copy. The same helper binds **`start: 'top bottom'`** → **`end: 'bottom top'`** so art drifts while the section crosses the viewport:

```ts
const bindPanelBgParallax = (sectionEl: HTMLElement | null, imgEl: HTMLElement | null) => {
  if (!sectionEl || !imgEl) return
  gsap.fromTo(
    imgEl,
    { x: 0, y: 0 },
    {
      xPercent: 0,
      yPercent: 25,
      ease: 'none',
      scrollTrigger: {
        trigger: sectionEl,
        start: 'top bottom',
        end: 'bottom top',
        scrub: true,
      },
    },
  )
}
```

### Reduced motion

When **`prefers-reduced-motion`** is set, **`useSectionsScrollExperience`** returns before creating tweens: **`gsap.set(sections, { clearProps: 'opacity,transform' })`**, then **`clearProps: 'transform'`** on **`.obsessions__scroll-shift`** and each mood/niche/newsletter **`img`**, so pinned/scrubbed transforms do not fight native scrolling.

### Alternate recipe: nested shell + `--oa-cover-p`

The preview below is **deliberately different**: a **fixed-height embed** owns its own scroll container. Reduced motion uses **`ScrollTrigger`** with **`scroller: shell`** to drive **`--oa-cover-p`**; the smooth path uses **Locomotive Scroll v5** (Lenis-backed) with **`scrollCallback`** and **GSAP’s ticker** as in [`WritingObjectArchiveScrollPreview.tsx`](src/components/writings/WritingObjectArchiveScrollPreview.tsx). The shell uses **`data-lenis-prevent`** so **site** Lenis does not swallow wheel events inside the demo.

```ts
ScrollTrigger.create({
  trigger: track,
  scroller: shell,
  start: 'top top',
  end: () => `+=${Math.max(32, shell.clientHeight)}`,
  scrub: 0,
  invalidateOnRefresh: true,
  onUpdate(self) {
    shell.style.setProperty('--oa-cover-p', String(self.progress))
  },
})
```

## Part 4 — Interior style rail: hover / focus swaps the panel (GSAP timeline)

**Target:** `.writing-oa-style__panel-inner` opacity and **`yPercent`**. **Driver:** **`onPointerEnter`** / **`onFocus`** per style row. **Runtime:** **`gsap.timeline`**: ease out → **`call`** to swap `img.src` and caption → **`fromTo`** in; **`gsap.killTweensOf(inner)`** before each new hover so stacked timelines do not fight.

From `WritingObjectArchiveInteriorStylePreview`:

```ts
gsap.killTweensOf(inner)

const tl = gsap.timeline({ defaults: { ease: 'power2.inOut' } })
tl.to(inner, { opacity: 0, yPercent: 5, duration: 0.14 }, 0)
tl.call(() => {
  writeDom(i)
})
tl.fromTo(
  inner,
  { yPercent: 9, opacity: 0.55 },
  { yPercent: 0, opacity: 1, duration: 0.42, ease: 'power2.out' },
)
```

Reduced motion: **`writeDom`** only and **`gsap.set(..., { clearProps: 'transform,opacity' })`**.

## Part 5 — Resources: idle list, then hover / focus swaps the portrait (instant, fixed slot)

**Target:** Same **`img`** element in a **fixed 3:4** box; caption and overlay title text. **Driver:** hover/focus on list buttons. **Runtime:** **immediate DOM** updates—no GSAP on the image. **`ul { display: contents }`** lets each **`li`** stack with a centered **`.writing-oa-res__float`** (`z-index: 2`, **`pointer-events: none`**) while the **active** row stays above the art; **`.is-idle`** hides float and caption dock until the first pick.

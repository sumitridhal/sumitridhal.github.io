## Part 1 — Hero curtain and split reveal

**Targets:** (1) full-bleed **curtain** layer sharing the stage box with the split media; (2) **split** halves (Étude de forêt left, Monet detail right) underneath; (3) **wordmark** overlay above both.

**From → to:** Curtain starts **closed** (covering the diptych) and tweens to **open** (`translateY` off the top of the stage). Split media stays mounted the whole time; wordmark does not leave the overlay stack—only opacity/contrast is implied by the scrim, not a separate position tween.

**Driver:** Initial sequence on load; **Replay intro** re-applies the same animation class on the curtain after a microtask so the browser can restart the keyframes.

**Runtime:** CSS `@keyframes` on the curtain (see `WritingObjectArchiveHeroRevealPreview` + `.writing-oa-hero*` in `_writing.scss`). Not GSAP in this embed.

**Replay / reverse:** Replay button retriggers the curtain cycle. There is no partial “scrub backward” in the preview—only a full replay.

**Reduced motion:** Preview adds a `--reduced` modifier class: curtain snaps open, **Replay** is disabled so we do not imply a timed experience.

## Part 2 — Left column: stacked plates and scale ladder

**Targets:** Sixteen absolutely stacked `<img>` nodes (one per plate: each slide’s **background** then **mid**, in order), each centered with `translate(-50%, -50%)` and animated **uniform scale** only. Right half of the stage is static Monet detail (no tween in this part).

**From → to:** State is indexed by `advanceCount` (0 … N−1). For layer index `j` at a given step:

- **Passed** layers (`j ≤ advanceCount`) tween toward **`BIG_SCALE`** — the “already revealed” zoom relative to the left panel (tuned in code so the plate reads full-bleed in the clip).
- **Focal** layer (`j === advanceCount + 1`, while it exists) sits at **`MID_SCALE`** — the current inset / readable plane.
- **Ahead** layers use **`HIDDEN_SCALE`** — effectively off-stage until the ladder reaches them.

Each **Next →** increments `advanceCount` and runs one **parallel** GSAP timeline: every layer’s `scale` tweens together (`power2.inOut`, fixed duration in code, **0** when reduced motion). **← Back** decrements `advanceCount` with the same timeline pattern. The badge is a readout of depth (1 … N).

**Driver:** Click on **Next →** / **← Back**; buttons are disabled while the timeline is active.

**Runtime:** GSAP `timeline` with all layer tweens at label time `0` (no stagger in the current embed).

**Replay / reverse:** **← Back** is the reverse driver step-by-step; there is no separate “reset stack” control.

**Reduced motion:** Duration collapsed to zero so scales snap to the new ladder.

## Part 3 — Composite: static Part 1 hero + “Current obsessions” scroll strip

**Targets:** (1) **Hero stack** — Part 1 split + wordmark, **sticky** inside a tall **hero-slot** so it stays on screen while the first viewport of shell scroll is consumed. (2) **Obsessions** — cream block **overlaps** the hero (`margin-top: -viewport`) and **`translateY`** tied to **`--oa-cover-p`** so it **slides up to cover** the hero as vertical scroll advances 0→1 over one shell-height. (3) **Intro + strip** — inner **`padding-top`** eases from a lower band toward the top over the same progress; strip still uses **`scrollLeft`** (`overflow-x: auto`). **`data-lenis-prevent`** on the shell so page-level Lenis ignores this region.

**From → to:** **`--oa-cover-p`** = **`min(1, scroll / shellHeight)`** where **`scroll`** is Lenis’s smoothed vertical offset (default path) or native **`scrollTop`** (reduced motion). Raise layer: `translateY(viewport * (1 - p))`. Inner intro: `padding-top` shrinks as `p → 1`. Horizontal strip unchanged in runtime.

**Driver:** Vertical wheel / trackpad on the shell (Locomotive / Lenis when motion is allowed). Horizontal gestures on `.writing-oa-scroll__viewport` when focused or direct.

**Runtime:** **Locomotive Scroll v5** with **`lenisOptions.wrapper`** = shell, **`content`** = track; **`scrollCallback`** → **`--oa-cover-p`**; **GSAP `ticker`** hosts Locomotive’s render loop. **`allowNestedScroll`** so the strip can scroll horizontally. **`ResizeObserver`** → **`loco.resize()`**. Reduced motion: **GSAP `ScrollTrigger`** on the track with **`scroller: shell`**, **`scrub: 0`**, **`onUpdate`** → **`--oa-cover-p`**, and **`ScrollTrigger.refresh()`** on resize. Strip: **`overflow-x: auto`**.

**Replay / reverse:** Scroll the shell up/down; strip scrolls independently.

**Reduced motion:** Intro **`padding-top`** choreography is disabled (`0`); cover **`translateY`** follows native **`scrollTop`** (no Locomotive instance).

## Part 4 — Interior style rail: hover / focus swaps the panel (GSAP timeline)

**Targets:** (1) **Kicker + list** — “Explore by interior style” label and a vertical **style list** (typographic active state: weight, italic, ink). (2) **Right panel** — portrait frame + monospace **caption** keyed to the **hovered or focused** style. (3) **Static stage** — no nested scroll; the block is a fixed-height **cream** panel inside the article.

**From → to:** **`onPointerEnter`** on each list row (and **`onFocus`** on the button for keyboard) calls **`goTo(index)`**. A **GSAP `timeline`** on **`.writing-oa-style__panel-inner`**: quick **out** (`opacity`, `yPercent`), **`call`** to swap **`img.src`** + caption text, then **fromTo** **in** so the stack reads as rising **from center toward rest**.

**Driver:** Pointer crossing a style row; keyboard **Tab** / **Shift+Tab** onto a style button.

**Runtime:** **GSAP `timeline`** (`to` → `call` → `fromTo`); **`gsap.killTweensOf`** before each new hover so rapid moves do not stack. No **`ScrollTrigger`** in this embed.

**Replay / reverse:** Move the pointer or focus to another line; the same timeline pattern runs again.

**Reduced motion:** No tween durations—**`writeDom`** only and **`clearProps`** on the panel inner.

## Part 5 — Resources: idle list, then hover / focus swaps the portrait (instant, fixed slot)

**Targets:** (1) **Kicker + list** — one **serif** list only. (2) **Overlap stack** — **`ul { display: contents }`**, centered **`.writing-oa-res__float`** over inactive rows (**`pointer-events: none`**), **active `z-index`** above the image. (3) **Portrait** — single **3:4** frame; **`object-fit: cover`** so every CDN still occupies the **same dimensions**. (4) **Caption** — **`.writing-oa-res__caption-dock`** with **`.is-idle`** until a pick.

**From → to:** **`onPointerEnter`** / **`onFocus`** → **`goTo(index)`** updates **`img.src`**, overlay title, and caption text **immediately** (no tween).

**Driver:** Pointer or keyboard focus on a line button.

**Runtime:** DOM + **CSS** only for visibility (**`.is-idle`** on float and caption dock). No GSAP in this embed.

**Replay / reverse:** Move across rows; same row is a no-op.

**Reduced motion:** N/A for motion (there is none on the image); layout unchanged.

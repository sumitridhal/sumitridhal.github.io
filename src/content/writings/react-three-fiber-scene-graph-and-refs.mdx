JSX in R3F does not recreate the DOM—it mirrors Three’s `Object3D` tree. Lowercase tags map to constructors (`<mesh>` → `THREE.Mesh`), and children become attach targets (`<boxGeometry />` attaches to `mesh.geometry`). When you need imperative Three APIs—BVH builds, `BufferAttribute` edits, loaders returning ready-made groups—refs and `primitive` are the bridge.

### `useRef` on declarative nodes

`useRef<THREE.Mesh>(null)` passed to `ref` on `<mesh>` resolves to the underlying `THREE.Mesh` after commit. That is the right handle for raycasts, `lookAt`, or swapping materials without re-mounting geometry.

```tsx
const meshRef = useRef<THREE.Mesh>(null)

useLayoutEffect(() => {
  meshRef.current?.geometry.computeVertexNormals()
}, [])
```

Prefer **`useLayoutEffect`** for one-time geometry fixes that must run before the browser paints the first WebGL frame; keep continuous updates in `useFrame`.

### `args` and the constructor

The `args` prop forwards to `new ThreeClass(...args)`. It is easy to overuse: changing `args` on every render can recreate heavy resources. For dimensions that change often, mutate the existing geometry or use a `key` to force a clean instance.

```tsx
<mesh>
  <cylinderGeometry args={[0.5, 0.5, 1.2, 32]} />
  <meshStandardMaterial color="#e0a080" />
</mesh>
```

### When to use `<primitive object={…} />`

Use `primitive` when you already have a `THREE.Object3D` from a loader, procedural builder, or external library. R3F will **not** dispose it automatically—you own the lifecycle.

```tsx
import { useMemo } from 'react'
import * as THREE from 'three'

function TorusKnotFromImperativeApi() {
  const mesh = useMemo(() => {
    const geo = new THREE.TorusKnotGeometry(0.45, 0.14, 160, 18)
    const mat = new THREE.MeshStandardMaterial({ color: '#88c0d0', roughness: 0.35 })
    return new THREE.Mesh(geo, mat)
  }, [])
  return <primitive object={mesh} />
}
```

The same pattern applies to `GLTFLoader` output: keep `gltf.scene` in state or a ref and mount `<primitive object={scene} />`, disposing cloned materials when you tear the asset down.

### Transforms vs React state

Position, rotation, and scale can be set declaratively (`<mesh position={[0, 1, 0]} />`) or imperatively in `useFrame`. Mixing both for the same axis in one frame causes confusion: pick one source of truth. High-frequency motion (physics, springs) usually belongs in refs and `useFrame`; rare layout changes belong in props.

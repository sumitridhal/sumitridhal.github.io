Believable spines need **CSS geometry first**—flex on a shared baseline, **full jacket spreads** cropped to a narrow column with `object-fit: cover`, and a few intentional `rotate()` degrees so the row feels human. GSAP then answers how the shelf **enters** and how each spine **responds**.

The preview is lazy-loaded so listing routes do not pay for GSAP until you open this note. Inside the stage, a short **`gsap.timeline()`** raises the one-line headline, then brings spines up with **`stagger: { each, from: 'center' }`** and `transformOrigin: '50% 100%'` so motion reads as lifting from the shelf. **`mouseenter` / `mouseleave`** fire small `gsap.to` calls with `overwrite: 'auto'` so fast cursor passes do not stack conflicting tweens.

```ts
const ctx = gsap.context(() => {
  const tl = gsap.timeline()
  tl.to(titleLine, { opacity: 1, y: 0, duration: 0.58, ease: 'power2.out' })
  tl.to(
    spines,
    {
      opacity: 1,
      y: 0,
      rotation: (_i, target) => Number(target.dataset.rotate ?? '0'),
      duration: 0.78,
      stagger: { each: 0.036, from: 'center' },
      ease: 'power3.out',
    },
    '-=0.32',
  )
}, rootRef)
```

```ts
const base = Number(spine.dataset.rotate ?? '0')
spine.addEventListener('mouseenter', () => {
  gsap.to(spine, { scaleY: 1.06, y: -12, rotation: 0, duration: 0.26, ease: 'power2.out', overwrite: 'auto' })
})
spine.addEventListener('mouseleave', () => {
  gsap.to(spine, { scaleY: 1, y: 0, rotation: base, duration: 0.34, ease: 'power2.inOut', overwrite: 'auto' })
})
```

Structure still benefits from boring layout primitives—here a flex row with `align-items: flex-end` and per-spine width tokens—so Claude Code can propose grid tracks or custom properties while you keep **art direction** (contrast, rhythm, how much lean is too much).

Claude Code shines when the feedback loop stays tight: tweak easing numbers, reload, paste a short screen recording or describe what still feels stiff. Let the model shuffle **stagger order** or suggest `gsap.context` boundaries; you hold the guardrails—**one clock** for the entrance (timeline), **short** hover tweens, and the same reduced-motion contract as the rest of the site (see [Driving Lenis from GSAP’s ticker instead of fighting two rAF loops](/writing/lenis-gsap-scroll-loop) for the broader “single rAF family” idea).

Ship a vertical slice first: a handful of books, one entrance pattern, one hover rule. Once motion and hierarchy read well, swap in real data and worry about horizontal scroll or virtualization. Vibe coding is not skipping engineering; it is letting polish lead as long as the foundations stay testable and reversible.
